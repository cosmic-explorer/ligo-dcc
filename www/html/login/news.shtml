Aug 22, 2013

A DCC upgrade focused on signatures, email notifications, short URL functionality, and user interface enhancements:

  • Permit the modification of a number of relatively innocuous document attributes without resetting a signature chain, e.g., Keywords, Topics, Related Documents, view/modify rules, Author list, and the Signature list itself. Note, changing other attributes will continue to reset any pre-existing signatures, e.g., Title, Abstract, Notes, and actual uploaded files.
  • Ditto for not resetting public access to documents when any of the "innocuous" attributes are changed.
  • For both the case of documents with signatures and public access, a second "do you really want to do this" confirmation will be made if a significant change has been requested that will reset the signature chain or public access. In other words, resetting a signature chain or public access to a document will require two actions by the user: 1) request one or more significant changes to a such a document; 2) enter a confirmation to proceed with the reset or select an option to cancel and review the significant changes.
  • The DCC now has better support for Latin1 (ISO8559-1) character set, which means that words (i.e., names) with special characters are now more often input and displayed correctly.
  • On the Preferences page, default authors can now be selected from a list, in addition to typing in names by hand.
  • On the Search page, it's now possible to deselect a name that's been highlighted in the drop-down list of authors.
  • Rules on using either a fuller, more inclusive full list of authors vs a smaller list of active authors have been clarified.
  • Signoff selection now supports name autocomplete and sorted list of names without opening a separate popup window.
  • It's now possible to search for documents by the name of the person that made most recent update (either a change of files or a change of metadata).
  • It's now possible to use LIGO identifiers when selecting document notification.
  • Email notifications by document type are now supported.
  • Email notifications now support the short URL format.
  • Email notifications now honor the same access rules as a document view.
  • When a final signature is applied to a document, an email notification is now sent to the original document Submitter, Authors, and everyone that has modified the current version, in addition to all the signers.
  • A more refined search for filenames with unusual characters (e.g., a pound sign).
  • Short URL feature has been expanded: Use it for filenames by adding a filename after a document identifier -- for example,
    https://dcc.ligo.org/LIGO-M080375/public/LIGO-M080375-V11 (Abbreviations And Acronyms).pdf;
    https://dcc.ligo.org/G1200837/Fractal analysis of LIGO s data.pdf
    Note that the feature works with or without the "LIGO-" prefix or a version number, and it works for both public and non-public documents.
  • It's now possible to specify "xml" or "html" as the output format when retrieving a document using a short URL:
    https://dcc.ligo.org/LIGO-G0900090/of=xml
    https://dcc.ligo.org/LIGO-G0900090/of=html
    If no output format value is specified, the default HTML format is used.
  • You can now click to email submitters, authors, editors, and signers of a document.
    A single email will be sent to the selected group, with the document identifier as the subject of the message.
  • Highlight the "Signature Report" button on the Home page, for users that have that option, when the report for the current user indicates there are one or more documents waiting for their signature.
  • Error messages are now disambiguated between unauthorized and non-existent document access attempts.
  • Enable all members of the DCC group Lab_Management to certify documents, thereby making public_pending documents actually become publicly accessible.
  • A large number of minor bug fixes

Mar 24, 2013

Viewing PDF files in Firefox version 19:

When viewing Engineering drawings, the Firefox PDF viewer does not show red lines in PDF files; SolidWorks drawings are rendered unreadable. This is a limitation of the browser's built-in PDF viewer. It is not a problem with the drawings or with the DCC. Disable the built-in viewer and use an external program such as Adobe or Preview.

Mar 01, 2013

DCC version 3.0.0 released with these features:

  • Autocomplete for author names:
    • Just typing a few characters will bring up a list of possible matches.
    • Choose if you want all authors or only active authors in the autocomplete, in the "Preferences" panel.
    • Works for a single author, in the front page, or
    • Works for multiple authors of a document, or selecting default authors, or searching for authors.
  • Federated identity: The DCC can be used by people who have an identity from a LIGO partner:
    • At the login portal, you can use your ligo.org identity, also called your "albert.einstein".
    • You can also login to DCC with an identity from the University of Tokyo, for LIGO affiliates who work for KAGRA.
    • You can also use your "LIGO Guest" identity to log in.
    • In the future, other partners will be added to this federation.
  • Short URLs: Quote your documents with simple, easy URLs:
    • Document links can be of the form https://dcc.ligo.org/M1000066.
    • Document links can be of the form https://dcc.ligo.org/LIGO-M1000066.
    • Explicit version numbers can be added, as in https://dcc.ligo.org/M1000066-v17.
    • Public documents can be quoted as in https://dcc.ligo.org/M1000066/public.
    • Specific files from a document can be quoted as in https://dcc.ligo.org/M1000066/pdf, so long as the result is unique.
  • Explicit permissions are now required for documents:
    • For example, a talk to the LSC might have view permissions as "LSC" and "Authors"; with modify permissions set to "Authors".
    • For more information, see https://dcc.ligo.org/M1000108
  • Redesigned portal page:
    • Restructured content makes the page easy to use
    • Improved design makes it visually appealing
For more information, see Whats New.

Mar 08, 2011

DCC version 2.3.0 was released with better usability for searching and author input. Among the new features:

  • Shortcuts to some frequent searches -- e.g., documents by the logged in author, and a search by title -- were added to the main page.
  • An updated author input field makes it easy to list authors in a particular order.
  • Basic search was fine-tuned: Since all words entered should no match, a number of returned matches is reduced. Also, the default ordering of returned documents was changed.
  • In the table of search results, column headings can now be sorted by clicking on the heading.
  • Advanced search interface, which gives greater control over search parameters, is now featured more prominently on the main page.
  • Files with names that contain nonstandard characters (eg, ";^") can no longer be uploaded.
  • Easily find who's in which access group: Select "List documents by Group" on the main page.
  • Default to only one Main file when creating a new document or version.
  • A longer list of bug fixes and code cleanup.

Oct 08, 2009

DCC version 2.2.2 released.

  • DCC was upgraded with a faster search engine, more precise default author and file content searches, and a richer group access hierarchy.
  • Since people usually search for "small author list" documents, default author searches now exclude matches against groups such as "LSC". To include all author groups in the search, use Advanced Search and select "Include Author Groups".
  • The simple text search now searches for documents with ALL, rather than ANY, substrings specified by the user.
  • The search engine is now faster. In addition, new documents are uploaded to the system in real-time, without a 30-min wait as before.
  • Additional hierarchial complexity has been added to the author group access atructure. Major author groups can now have sub-groups that inherit membership of the parent group. For example, aLIGO Business group inherits access rights of its parent Lab Business group.
  • Documents that are publicly accessible now have a prominent link to their public version in the upper-left corner of the page that describes the document, or its document card. This allows easier access to public documents by external collaborators.

May 07, 2009

DCC version 2.2.0 released.

  • The upgrades allows to select large groups of authors at once, e.g., for a paper by the LSC or Virgo Collaboration, in addition to individual authors. This allows to search for a document by either a particular author, or by the author's group of affiliation. For example, a search for an LSC member will return all documents with LSC as an author group; to search for a particular LSC member's documents instead, de-select "Include Author Groups" in your search.
  • Additional access groups not in the original release have been added.
  • A few bug fixes have been made.

Jan 26, 2009

The new DCC was released to the LSC and Virgo Collaborations. The new system has a substantially expanded set of features, which include:

  • A structured hierarchy of group access privileges, from Lab Directorate (with the broadest access) to Public (with the narrowest).
  • An information-rich description ("metadata") of documents, which allows for highly selective searches.
  • A detailed user interafce that allows users to create documents, upload files, and make edits to existing documents.
  • An option of receiving e-mail notifications of changes to a document of your interest.
  • An option of grouping documents associated with a particular event.
